# 井字遊戲 Tic-Tac-Toe 🎮

一款以 Vanilla JavaScript + Node.js 打造的網頁版井字遊戲，支援 **AI 對戰**（簡單／困難）與 **線上多人即時對奕**。

---

## 功能特色

| 功能 | 說明 |
|------|------|
| 🤖 AI 簡單模式 | AI 從空格中隨機選擇落子位置，適合新手練習 |
| 🧠 AI 困難模式 | AI 使用 Minimax 演算法計算最佳落子，幾乎無法擊敗 |
| 🌐 線上多人對奕 | 透過公開大廳建立或加入房間，與真人即時對戰 |
| 🏆 勝利高亮 | 遊戲結束時自動高亮三個獲勝格子 |
| 🔄 再來一局 | 雙方同意後立即重置棋盤，無需重新整理頁面 |
| 📱 響應式設計 | 棋盤在手機與桌機上皆保持正方形並置中 |
| ♿ 無障礙支援 | 所有格子附有 `aria-label`，支援鍵盤操作 |

---

## 技術棧

- **前端**：HTML5、CSS3（Flexbox / Grid、CSS Variables）、Vanilla JavaScript (ES6+)
- **後端**：Node.js + Express（靜態檔案服務）
- **即時通訊**：Socket.io（WebSocket）

## 遊戲流程

```
輸入暱稱
    │
    ▼
選擇對戰模式
    ├── 🤖 AI 對戰（簡單）  ──→ 直接開始遊戲（本地模式）
    ├── 🧠 AI 對戰（困難）  ──→ 直接開始遊戲（本地模式）
    └── 🌐 線上多人
            │
            ▼
       公開大廳
            ├── 建立房間 ──→ 等待對手加入（顯示 4 碼房間代碼）
            └── 加入房間 ──→ 雙方進入遊戲
```

---

## 線上多人模式說明

1. **建立房間**：點擊「＋ 建立房間」，系統自動產生 4 碼代碼（例如 `K7MN`）
2. **等待對手**：畫面顯示代碼，可點擊代碼複製並分享給朋友
3. **加入房間**：在公開大廳點擊任意等待中的房間即可加入
4. **再來一局**：遊戲結束後，雙方點擊「再來一局」即可立即重賽
5. **斷線處理**：若對手斷線，系統會即時通知另一方並回到大廳

## 棋盤格子索引

```
 0 | 1 | 2
-----------
 3 | 4 | 5
-----------
 6 | 7 | 8
```

獲勝條件（共 8 條）：
- 橫排：`[0,1,2]`、`[3,4,5]`、`[6,7,8]`
- 直行：`[0,3,6]`、`[1,4,7]`、`[2,5,8]`
- 對角：`[0,4,8]`、`[2,4,6]`

## Socket.io 事件一覽

| 方向 | 事件名稱 | 說明 |
|------|----------|------|
| 客→服 | `enter_lobby` | 進入公開大廳 |
| 客→服 | `create_room` | 建立新房間 |
| 客→服 | `join_room` | 加入指定房間 |
| 客→服 | `make_move` | 玩家落子 |
| 客→服 | `request_rematch` | 投票再來一局 |
| 服→客 | `room_list` | 廣播公開房間清單 |
| 服→客 | `room_created` | 建立房間成功（含代碼） |
| 服→客 | `game_start` | 遊戲開始（含分配棋子） |
| 服→客 | `game_update` | 棋盤更新（含勝負判斷） |
| 服→客 | `rematch_start` | 雙方同意再來一局 |
| 服→客 | `rematch_requested` | 對手想再玩 |
| 服→客 | `opponent_left` | 對手已離開 |
| 服→客 | `join_error` | 加入房間失敗 |
